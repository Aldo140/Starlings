Don't insert animations as decorations bolted onto existing sections. Each animation must be the **defining visual character** of its section — design the section around what the animation communicates.

**Why:** After implementing all 6 Uiverse animations in an initial pass, the user rejected every placement: *"i dont like the use of any of the new additions study them see where they fit best create design around that."* The failure was treating animations as decorations (e.g., a 300×150 side panel for the flashlight) rather than letting the animation's emotional character define the section.

**How to apply:** Before placing any animation, ask: *what does this animation communicate emotionally or semantically?* Then design a section where that communication IS the section's purpose — not a footnote. The animation should answer "what is this section about?" on its own.

Examples of the correct approach (from the v2 redesign):
- Flashlight (city/discovery) → fills the ENTIRE Community Partners header. You can't miss it.
- Book page-flip (knowledge/gathering) → IS the MurmurationSyncBanner's primary visual. The metaphor lands.
- 3D cards (depth/premium) → every partner IS a 3D card, not just one featured CTA.
- Bouncing cube (playful loading) → floating pill companion to the loading bar, shows up site-wide.
